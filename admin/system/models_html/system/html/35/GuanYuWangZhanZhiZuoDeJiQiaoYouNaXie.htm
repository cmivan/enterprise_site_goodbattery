一、样式表的使用
我们在设计过程中，没有考虑到css样式表的整体安排。网站是三个人制作的，每个人负责一部分网页，都定义了各自的样式表。等我们把网页整合到一起时，发现样式全乱了。两个人定义的样式表混乱在一起。我们只好重新进行样式定义，增加了很多的工作量。
所以，下面我就总结一下样式表的使用。
CSS样式使用，有以下四种方法：使用<link ...>链接到外部样式表，使用@import引入外部样式表（这种方法会将样式表复制到当前位置，这样浪费资源，所以我们暂时不考虑），直接用<style>...</style>嵌入样式表，内联样式表。这几种方法该怎么使用呢？
我们使用外部调用法，好处不言而喻，只需修改.css文件就能改变页面的样式。如果所有页面网站制作网站设计 都调用同一个样式表文件，那么改一个样式表文件，可以改变所有文件的样式。除此之外，会使用户的浏览速度加快。因为当用户第一次打开链接了样式表的页面后，样式表会自动下载到本地缓存当中。当用户再打开另一个页面，而这个页面如果也链接了同样的样式表的时候，会从缓存中读取相关的文件，这样就会加快浏览的速度。
但是碰到网络不顺畅的情况，页面打开以后，链接的样式表没有加载成功。这种情况造成的后果，不仅仅是页面文字的颜色、大小等等样式无法显示，更麻烦的是很可能造成整个页面的布局乱了套！
所以个人认为：首页完全采用嵌入的方法，其他页面均采用链接的方式。因为首页是一个网站流览量最大的页面，必须保证高的可靠性，这样可以保证首页在网络状况不好的情况下仍然能够正确显示，同时也能保证修改比较方便。
此外，还有一种补充方案，就是对于各个页面都会用到的样式：采用链接。对于只有这一个页面会用到，但会用到很多次的样式：采用嵌入。而对于只有这一个页面会用到且只使用一次的样式可以采用内联样式。
　

二、网站中插入视频
我们有两种方案，一种是弹出式，需要建构流媒体服务器并且需要绝对IP地址；另一种是采用嵌入式，只需相对位置。考虑到稳定性和方便性，我们决定采用嵌入式，具体代码如下：
<table width="328" border="0" cellspacing="0" cellpadding="0">
<tr>
<td width="328"><object width="320" height="250" classid="clsid:CFCDAA03-8BE4-11cf-B84B-0020AFBBCCFA">
<param name="CONTROLS" ="ImageWindow">
<param name="CONSOLE" ="Video">
<param name="CENTER" ="TRUE">
<param name="MAINTAINSPECT" ="TRUE">
</object></td>
</tr>
<tr>
<td><object width="320" height="30" classid="clsid:CFCDAA03-8BE4-11cf-B84B-0020AFBBCCFA">
<param name="CONTROLS" ="StatusBar">
<param name="CONSOLE" ="Video">
</object></td>
</tr>
<tr>
<td><object width="320" height="30" classid="clsid:CFCDAA03-8BE4-11cf-B84B-0020AFBBCCFA">
<param name="CONTROLS" ="ControlPanel">
<param name="CONSOLE" ="Video">
<param name="SRC" ="/being_movie/J9.rm">
<param name="AUTOSTART" ="TRUE">
<param name="PREFETCH" ="0">
<param name="LOOP" ="0">
<param name="NUMLOOP" ="0">
</object></td>
</tr>
</table>
